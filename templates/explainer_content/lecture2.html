رجوع فتح في تبويب جديد

أهلاً بك! بناءً على المحاضرة الثانية

سنتناول مفاهيم أساسية في C++، مع التركيز على الجمل الشرطية وجمل التكرار، بالإضافة إلى كيفية التعامل مع الـ char كنوع بيانات. سأحرص على استخدام المصطلحات البرمجية بالإنجليزية لضمان الوضوح، مع توفير أمثلة كود وشرح للنقاط الهامة التي ذكر المحاضر أنها قد تكون جزءًا من الامتحان.

التحكم في سير البرنامج (Control Statements)

البرنامج ليس مجرد عمليات جمع وطرح وطباعة نتائج، بل غالبًا ما تحتاج إلى توجيه سيره لاتخاذ قرارات أو تكرار عمليات. هذا هو دور Control Statements.

الجمل الشرطية (Selection, Conditional, or Branching Statements)

هذه الجمل تتيح لك اختيار مسار معين بناءً على شرط.

1. جملة if
  • تسمح لك بتنفيذ كتلة معينة من الكود فقط إذا كان الشرط صحيحًا (قيمة غير صفرية - Non-Zero).
  • تقييم الشرط: تعتمد جملة if على تقييم القيمة التي تليها. إذا كانت القيمة صفر (Zero)، فإن الشرط يعتبر خاطئًا. إذا كانت القيمة غير صفرية (Non-Zero)، فإن الشرط يعتبر صحيحًا. لا يهم مصدر هذه القيمة (سواء كانت ناتجة عن عملية منطقية، حسابية، أو غيرها).
  • أهمية الأقواس (()): الأقواس هي ذات الأولوية الأعلى (Highest Priority) ويجب استخدامها دائما لتنظيم العمليات داخل الشرط، مما يساعد على الوضوح وتجنب الأخطاء.
  • مثال: التحقق من عدد فردي أو زوجي.
    • تحسين الشرط: يمكن تبسيط الشرط if (num % 2 != 0) إلى if (num % 2). ذلك لأن نتيجة num % 2 ستكون 1 (قيمة غير صفرية، تعتبر true) للأعداد الفردية، و 0 (تعتبر false) للأعداد الزوجية.
2. جملة if...else
  • تستخدم عندما يكون لديك مساران محتملان للبرنامج. يتم تنفيذ كتلة واحدة فقط: إما كتلة if (إذا كان الشرط صحيحًا) أو كتلة else (إذا كان الشرط خاطئًا).
  • مثال: تقييم درجة طالب (ناجح/راسب)
int grade = 75;
if (grade >= 60) {
    cout << "Pass"; // This will be printed
} else {
    cout << "Fail";
}
3. جملة if...else if المتداخلة (Nested if / else if)

تستخدم عندما يكون لديك أكثر من مسارين محتملين، ويتم تقييم الشروط بالتسلسل.

  • المشكلة الأصلية: عند محاولة تحديد الدرجات (A, B, C) بناءً على نطاقات متعددة، قد يخطئ المبرمج في فهم الشروط الضمنية.
  • سوء الفهم الأولي: قد يظن المبرمج أن if (grade >= 80) تعني ضمنيًا "إذا كانت الدرجة بين 80 و 90". لكن هذا ليس صحيحًا، فـ 95 هي أيضًا أكبر من 80.
  • الحل الأمثل (else if المتسلسل): هذا هو الحل المفضل والأكثر كفاءة، حيث يتم وضع الشروط المتتالية داخل كتل else، مما يضمن أن يتم تنفيذ كتلة واحدة فقط بمجرد تحقق الشرط.
    مثال: إذا كانت الدرجة 95، يدخل في الشرط الأول (grade >= 90)، يطبع "A"، ثم يتخطى جميع كتل else المتبقية. 
int grade = 85;
if (grade >= 90) {
    cout << "A";
} else if (grade >= 80) {
    cout << "B"; // This will be printed
} else if (grade >= 60) {
    cout << "C";
} else {
    cout << "F";
}
ملاحظة على الأقواس المعقوفة {}: إذا كانت كتلة الكود داخل if أو else تحتوي على Statement واحد فقط، فيمكنك الاستغناء عن الأقواس. ومع ذلك، يفضل استخدامها دائمًا للحفاظ على وضوح الكود وتجنب الأخطاء.

جملة switch

تستخدم عندما يكون لديك شرط يعتمد على مقارنة متغير واحد بقيم ثابتة ومحددة (Constants).

  • شروط الاستخدام: يفضل استخدام switch عندما تتحقق الشروط التالية:
    1. التحقق من المساواة (Check for Equality).
    2. التحقق من قيم ثابتة (Check against Constants).
    3. التحقق من متغير من عائلة Integer (مثل int, char, short, long). لا يمكن استخدام float, double, أو string.
    4. العمليات منفصلة لكل حالة (Separate Operations).
  • أهمية break: عند دخول case معين، يستمر البرنامج في تنفيذ جميع الـ cases التالية (Fall Through) حتى يجد كلمة break أو يصل إلى نهاية switch. وظيفة break هي إنهاء الـ switch statement والخروج منها بالكامل.
  • قسم default: قسم اختياري يتم تنفيذه إذا لم تتطابق قيمة المتغير مع أي من الـ constants المحددة في الـ cases.
ملاحظة هامة: المحاضر شدد على أن break يجب أن تستخدم فقط عندما تكون ضرورية جدًا، مثل داخل switch. لا ينصح باستخدامها بشكل عام للخروج من الـ function أو الـ loop إذا كان هناك حلول بديلة باستخدام الـ control flow الطبيعي (مثل شروط if)، لأنها قد تجعل الكود أقل وضوحًا وأصعب في التتبع.

جمل التكرار (Iteration Statements or Looping Statements)

تستخدم لتكرار مجموعة من الأوامر عدة مرات.

1. جملة for

تستخدم غالبًا عندما تعرف عدد مرات التكرار مسبقًا.

  • خطوات التنفيذ:
    1. Initialization Statement: يتم تنفيذها مرة واحدة فقط عند بداية الـ loop.
    2. Condition Statement: يتم تقييمها في كل تكرار. إذا كانت صحيحة، يستمر التكرار.
    3. Code block: إذا كان الشرط صحيحًا، يتم تنفيذ الكود داخل الـ loop.
    4. Update Statement: يتم تنفيذها بعد كل تكرار.
  • نقطة هامة (للامتحان): تجنب الـ Infinite Loop (مثل for(;;)) إلا في حالات محددة جدًا، مثل تطبيقات IoT أو الـ multi-threading. استخدامها غالبًا ما يشير إلى تصميم غير جيد للكود.
2. جملة while

تستخدم عندما لا تعرف عدد مرات التكرار مسبقًا، حيث يستمر التكرار طالما أن الشرط صحيح.

  • مقارنة بـ for: من الناحية الفنية، يمكن استبدال أي منهما بالآخر. لكن يفضل استخدام for عندما يكون عدد التكرارات معروفًا، و while عندما يعتمد التكرار على شرط معين قد يتغير أثناء التشغيل.
3. جملة do...while

تشبه while، لكنها تضمن تنفيذ كتلة الكود مرة واحدة على الأقل قبل التحقق من الشرط.

نقطة هامة (للامتحان): الـ do...while loop هي الوحيدة التي تضمن تنفيذ الكود مرة واحدة على الأقل. 
// مثال: قائمة اختيار (Menu) تتطلب إدخال صحيح
int choice;
do {
    cout << "1. Start Game" << endl;
    cout << "2. Options" << endl;
    cout << "3. Exit" << endl;
    cout << "Enter your choice: ";
    cin >> choice;
} while (choice < 1 || choice > 3);

التعامل مع الـ Characters

في C++، يتم تخزين الـ char (الحروف) في الذاكرة كأرقام، وذلك باستخدام نظام ترميز ASCII. كل حرف أو رمز على الكيبورد له قيمة رقمية مقابلة (مثال: 'A' يقابله 65، 'a' يقابله 97). لهذا السبب يمكنك معاملة الـ char كـ integer والعكس.

التعامل مع إدخال الـ char والمفاتيح الموسعة
  • المشكلة مع std::cin: عند قراءة حرف، فإنه ينتظر ضغط Enter. عند التعامل مع المفاتيح الموسعة (مثل مفاتيح الأسهم)، التي ترسل بايتين، يقرأ std::cin البايت الأول فقط ويتجاهل الثاني، مما يؤدي إلى فقدان المعلومة.
  • الحل الأفضل (getch و getche):
    • هذه الـ functions (موجودة في مكتبة conio.h) لا تنتظر ضغط Enter وتقرأ الحرف فورًا.
    • getch(): لا تطبع الحرف على الشاشة (No Echo).
    • getche(): تطبع الحرف على الشاشة (Echo).
    • الميزة مع المفاتيح الموسعة: نظرًا لأنها لا تمسح الـ buffer، يمكنك استدعاؤها مرة أخرى لقراءة البايت الثاني (الكود الفعلي للمفتاح الموسع) بعد قراءة البايت الأول (الذي يكون عادةً null character بقيمة 0).
نقطة هامة (للامتحان): معرفة كيفية التعامل مع المفاتيح الموسعة أمر ضروري. يجب التدرب على معرفة قيم ASCII للمفاتيح المختلفة مثل الأسهم، Delete، Backspace.

نقاط هامة وتركيز الامتحان

المحاضر شدد على عدة نقاط يجب التركيز عليها:

  • قراءة الأخطاء (Errors): يجب قراءة الأخطاء التي تظهر أثناء الـ compilation والـ runtime وفهمها جيدًا. الأخطاء النحوية (Syntax Errors) تكون واضحة، أما الأخطاء المنطقية (Logical Errors) فهي أصعب وتحدث بسبب خطأ في تصميم الخوارزمية.
  • أولوية العمليات والأقواس: استخدم الأقواس دائمًا لتحديد أولويات العمليات بوضوح لتجنب الأخطاء.
  • لا يوجد "فارغ" في الذاكرة: أي متغير يتم تعريفه في الذاكرة يحمل قيمة ما، إما تكون garbage value (قيمة عشوائية) إذا لم يتم تهيئته، أو القيمة التي أسندت إليه.
  • سرعة الكود واستهلاك الذاكرة: شكل الكود الأقصر أو الأكثر جمالًا ليس بالضرورة هو الأسرع أو الأقل استهلاكًا للذاكرة. استخدام الـ Macro بدلاً من const يؤدي إلى كود أسرع ويستهلك ذاكرة أقل.
  • عملية الـ compilation و linking: فهم المراحل التي يمر بها الكود من source code إلى executable file.
